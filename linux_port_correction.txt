CORRECTION: PGD3D must remain GPU-accelerated on Linux — do NOT implement a CPU fallback.

The correct cross-platform approach for PGD3D is to use Vulkan compute shaders instead of DirectX11 compute shaders on Linux.

Strategy:
- On Windows: keep the existing DirectX11 path exactly as-is
- On Linux: implement Vulkan compute shaders (using the Vulkan SDK / vulkan-headers + spirv-cross or glslang to compile GLSL equivalents of the 4 HLSL compute shaders)

The 4 HLSL shaders (CountAlphaValues, ParallaxToCM, ConvertToHDR, SSSFix) need to be rewritten as GLSL compute shaders (.comp files) and compiled to SPIR-V at build time or runtime.

The PGD3D class public interface stays the same, but the Linux implementation uses:
- VkDevice, VkQueue, VkCommandBuffer instead of ID3D11Device/ID3D11DeviceContext
- VkDescriptorSet / VkDescriptorSetLayout instead of SRV/UAV
- VkComputePipeline with SPIR-V shaders instead of D3D11ComputeShader
- VkBuffer for structured buffers, VkImage for textures
- For DDS loading/metadata: DirectXTex software path (no D3D11 required for that part)

The PGD3D::initGPU() on Linux should enumerate Vulkan physical devices and pick a suitable one (preferring discrete GPU, falling back to integrated).

Add `vulkan` as a vcpkg dependency for Linux:
```json
{ "name": "vulkan", "platform": "!windows" },
{ "name": "vulkan-memory-allocator", "platform": "!windows" }
```

Also add glslang (for shader compilation) or ship pre-compiled SPIR-V.

For the GLSL shaders, create equivalent .comp files in PGLib/cshaders/:
- CountAlphaValues.comp
- ParallaxToCM.comp
- ConvertToHDR.comp
- SSSFix.comp

These are 1:1 translations of the HLSL shaders to GLSL compute shaders.
